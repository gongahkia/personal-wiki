Lambda calculus

Abstract notation to describe functions and application, providing the basis for functional programming.

Introduction

  • written as λ\lambdaλ-calculus
  • created by Alonzo Church
  • can represent any turing machine
  • by definition the world's smallest programming language

Quickstart

Datatypes

λ\lambdaλ-calculus has 3 foundational datatypes.

  1. Variable
    • <name>
  2. Function
    • λ\lambdaλ <parameter>.<body>
    • function parameter and body are . period-delimited
    • functions are single parameter
  3. Application
    • <function><variable OR function>

In practice, it looks like this.

Example Explanation
xxx xxx is a variable
λx.x\lambda x . xλx.x function definition for a function with parameter xxx and body xxx
(λx.x)a(\lambda x . x)a(λx.x)a function call for the previously declared function λx.x\lambda x . xλx.x with the argument aaa

Free and Bound variables

  • Free variable: variable that is never declared prior to its specification in a function body
  • Bound variable: variable that is declared within both the function parameter and body

In practice, they look like this.

Example Explanation
λx.x\lambda x . xλx.x xxx is a bound variable
λx.y\lambda x . yλx.y yyy is a free variable

Evaluation

  • application evaluation is performed via β\betaβ-reduction
  • β\betaβ-reduction: lexically-scoped substitution, a concept most of us are familiar with from foundational mathematical functions
  • allows for higher-order functions, where functions themselves are provided as arguments to another function

In practice, it looks like this.

example 1

  • (λx.x)a(\lambda x . x)a(λx.x)a evaluates to aaa

example 2

  • (λx.y)a(\lambda x . y)a(λx.y)a evaluates to yyy

example 3

  • (λx.(λy.x))a(\lambda x . (\lambda y . x ))a(λx.(λy.x))a evaluates to λy.a\lambda y . aλy.a
  • this is an example of a higher-order function evaluated using β\betaβ-reduction
