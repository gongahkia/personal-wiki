Full-stack

To know anything, you must know everything.

For clarity, this document is about full-stack architecture.

Definitions

Within the context of software development.

  1. Architecture: blueprint of how a website or applicationâ€™s components interact with each other AND how data flows through the system
  2. Full-stack: consists of the three below layers
    1. Frontend (client-side code): UI and UX
    2. Backend (server-side code): logic, data processing, API handling
    3. Database (database code): for data storage and management
  3. API (application programming interface): bridge between the frontend and backend that defines communication via HTTP requests

Frontend

  • Primary user interface (UI) that directly impacts the user experience (UX)
  • Built using web technologies
    • Web programming
      • HTML
      • CSS
      • JavaScript or TypeScript
    • Frameworks (libraries)
      • Svelte
      • React
      • Vue
      • Angular
      • Solid
      • Astro
      • Vite
      • Next
      • Mithril
      • Preact
      • Ruby on Rails

Backend

  • Responsible for codification of business logic, database interaction, authentication and handling API requests
  • Built using backend technologies
    • Business logic
      • Python
      • JavaScript
      • Ruby
      • Java
      • C#
      • Go
      • PHP
    • Database interaction
      • Django
      • Flask
      • Node.js
      • Ruby on Rails
      • ASP.NET
      • Spring
      • Micronaut
      • Go
      • PHP
    • Authentication
      • JWT (JSON Web Token)
      • OAuth 2.0
      • OpenID Connect
      • SAML (Security Assertion Markup Language)
      • Session-based authentication
    • API handling
      • REST API
      • GraphQL
      • gRPC
      • WebSocket
      • Postman
      • Swagger

Database

  • Stores the data required by the application
  • Built using database technologies
    • SQL (relational databases)
      • MySQL
      • PostgreSQL
      • SQLite
      • MariaDB
      • IBM Db2
      • Google Cloud SQL
    • NoSQL (document-based databases)
      • MongoDB
      • Cassandra
      • Firebase
      • DynamoDB
      • CouchDB
